# meet.u — CPU 온도 (신뢰도) 기능 정의서

> 담당: 백엔드/프론트엔드/디자인 팀
> 범위: 사용자 프로필에 표시되는 **신뢰도 시각 지표** "CPU 온도" 기능
> 분리 이유: 본 기능은 **AI 매칭 알고리즘에 반영되지 않음**. 별도 문서로 정리하여 AI 모듈 명세에서 제외.

---

## 1. 개요

### 1.1 목적
프로젝트 참여 이력에 기반한 **사용자 신뢰도**를 직관적인 "온도" 메타포로 시각화한다. 다른 사용자가 팀원 선택 시 참고할 수 있는 보조 정보.

### 1.2 핵심 원칙
- **시각 참고용만** — AI 매칭 알고리즘에는 입력되지 않음
- **자동 계산** — 사용자가 직접 점수를 매기지 않고, 후기 태그를 기반으로 시스템이 환산
- **객관성** — 평가는 함께 작업한 팀원 간 상호 평가로만 변동

---

## 2. 데이터 정의

| 항목 | 값 |
|------|-----|
| 기본값 (가입 시) | **50°C** |
| 최저 값 | 30°C |
| 최고 값 | 99°C |
| 단위 표기 | "°C" (섭씨 기호) — 시각 메타포일 뿐 실제 온도 아님 |
| 갱신 시점 | **프로젝트 완료 시점** (팀원 상호 평가 완료 후) |
| 저장 위치 | `USER_TRUST` 테이블 (스키마는 DB 팀 담당) |

---

## 3. 후기 태그 정의

프로젝트 완료 후 각 팀원은 함께한 모든 팀원에 대해 다음 5개 태그 중 **1개를 필수 선택**.

| 태그 | 구분 | CPU 온도 변동 |
|------|------|---------------|
| 소통이 원활했어요 | 긍정 | +α |
| 기여도가 높았어요 | 긍정 | +α |
| 보통이었어요 | 중립 | 0 |
| 참여가 저조했어요 | 부정 | -β |
| 소통이 어려웠어요 | 부정 | -β |

> α, β 값은 백엔드/기획 협의로 확정 (예: α=+2, β=-2). 본 문서에서는 미확정 상태로 둠.

## 4. 기능 요구사항 (FR)

### FR-TRUST-01. 초기 값 설정
- 회원 가입 시 `USER_TRUST.value = 50`으로 초기화
- 신규 회원은 모든 화면에서 50°C로 표시

### FR-TRUST-02. 후기 평가 입력
- 프로젝트 상태가 "완료"로 전환되면, 각 팀원에게 **상호 평가 요청** 알림 발송
- 평가 화면: 같이 일한 팀원 N명에 대해 각각 태그 1개 선택
- **모든 팀원이 평가를 마쳐야 프로젝트가 진짜 "완료" 처리됨** (PEER_REVIEW 미입력 시 보류)

### FR-TRUST-03. 평가 기한
- 프로젝트 완료 처리 시점부터 **14일**
- 기한 초과 시 미평가는 **0점(중립) 자동 처리** — 해당 평가자가 받은 평가도 동일 처리
- 미평가 시 본인의 후속 프로젝트 평가 기능 일시 제한 (선택 사항, 정책 결정 필요)
- 협업 의지 평가(`ai_module_prd_srs.md` 6.1 참조)와 **동일 14일 기한**을 공유하지만, 두 평가는 **별개 데이터**:
  - 협업 의지 (+2/0/-1) → `COOPERATION_WILLINGNESS` → AI 매칭의 r_ii'
  - CPU 온도 후기 태그 (5종) → `PEER_REVIEW` → CPU 온도 점수 (AI 미반영)
- 평가 화면에서 두 가지를 같이 입력받을 수 있으나, 저장 위치와 후속 처리는 완전히 분리됨.

### FR-TRUST-04. CPU 온도 갱신
- 모든 팀원의 평가가 입력되면(또는 기한 도래 시점) 다음 식으로 갱신:
  ```
  new_value = clamp(current_value + Σ(tag_score), 30, 99)
  ```
  - tag_score: 본인이 받은 태그들의 점수 합산
  - clamp: 범위를 30~99로 강제

### FR-TRUST-05. 프로필 표시
- 마이페이지·다른 사용자 프로필 페이지에 CPU 온도를 표시
- 시각 표현: 온도계 아이콘 또는 숫자 + 색상 그라데이션 (디자인 팀 결정)
- 색상 구간(권장): 30~49 = 파랑(차가움), 50~69 = 노랑(보통), 70~99 = 빨강(뜨거움)

### FR-TRUST-06. 정렬·필터 미제공
- 사용자 검색에서 CPU 온도로 **정렬·필터링은 제공하지 않음**
- 이유: 신규 회원·미경험자가 구조적으로 불리해지는 것을 방지

## 5. 화면 기능 명세

### 화면 T-01. 프로젝트 완료 시 평가 화면
| 항목 | 내용 |
|------|------|
| 트리거 | 팀장이 프로젝트 "완료" 처리 |
| 표시 | 함께한 팀원 N명에 대해 태그 5개 라디오 버튼 |
| 액션 | "제출" 클릭 시 PEER_REVIEW 저장 |
| 사전 조건 | 모든 팀원 평가 미입력 시 프로젝트 완료 안 됨 안내 |

### 화면 T-02. 프로필 — CPU 온도 표시
| 위치 | 내용 |
|------|------|
| 본인 마이페이지 | CPU 온도 + 직전 갱신 시점 |
| 타 사용자 프로필 | CPU 온도만 (변동 이력 비공개) |
| 모집글 지원자 카드 | 닉네임·포지션·CPU 온도 (검색·필터 대상 아님) |

---

## 6. 비기능/제약

- **AI 매칭 알고리즘에 입력되지 않음**. RL-GP나 IFN 계산과 무관
- **변동 이력 공개 범위**: 본인만 조회 가능 (다른 사용자는 현재값만)
- **악의적 평가 방지**: 동일 프로젝트의 동일 팀원에 대한 평가는 1회만 (수정 불가)
- **데이터 보존**: 회원 탈퇴 시 USER_TRUST 즉시 삭제. PEER_REVIEW는 해시화 후 보관 또는 삭제 (개인정보 정책에 따름)
